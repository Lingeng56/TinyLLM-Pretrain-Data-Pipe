# 00 工程骨架与标准（Guidance）

**要教会你的知识点**
- 工业级项目的目录分层与职责边界
- 数据 schema 设计原则（可扩展、可验证、可追溯）
- 配置驱动与运行清单（manifest）的价值

**你需要完成的事情**
- 建立标准目录结构
- 定义统一 jsonl schema
- 编写 `configs/pipeline.yaml` 初版
- 定义 `reports/run_manifest.json` 模板

**建议步骤**
1) 创建目录：`configs/ pipelines/ connectors/ processors/ validators/ metrics/ packers/ reports/ data/ logs/ scripts/`
2) 设计 jsonl schema（字段、类型、必填/可选）
3) 编写 `pipeline.yaml`（路径、阈值、并发、采样策略）
4) 设计 run manifest（run_id、数据版本、输入输出、规则版本）

**验收标准**
- 目录齐全且职责清晰
- schema 文档可读且能覆盖后续需求
- `pipeline.yaml` 能加载并被程序读取
